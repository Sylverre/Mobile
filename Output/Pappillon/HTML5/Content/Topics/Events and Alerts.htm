System Notifications: Events and Alerts

CenterScape can notify users or outside systems whenever certain conditions are met. Administrators can configure Event and Alert notifications about states of the system, which includes hardware used to monitor it. Users with the appropriate permissions can create Event and Alert notifications about the state of the assets being managed and the environment being monitored.

Alerts use a set condition and clear condition, and an alert remains active until the clear condition is met. Events have defined triggers, and an event occurs each time the trigger condition is met. Events might be used to send data from CenterScape to other software systems or to notify users when certain conditions occur, while the Alerts are configured to ensure that you know the location of your assets, their operational state, and conditions of the environment in which they are being used.

Both events and alerts perform an assigned Action when conditions are met. Event and Alert Actions include sending email, posting messages via HTTP, writing notification messages to logs, etc. Alerts can also be sent to Serial Devices.

Create New Event or Alert

1. Set Threshold (Alerts) or Trigger (Events).

2. Create or select Action and Test.

To create a new Event, navigate to Admin Console > Events or User Console > Events. Set a new Event Trigger and assign or create an Event Action.

 

To create a new Alert, navigate to Admin Console > Alert Management or User Console > Alert Management. Set the Thresholds and assign or create an Alert Action.

 

Alerts for asset status and environmental conditions are set using the User Console. These conditions include Asset Offline, Differential Pressure, Door Open and Close, Fluid Detected, Humidity, Low Battery, Motion, Tamper, and Temperature conditions, plus Custom and Adaptive Alerts set by the user. The Admin Console is used to configure Alert condition filters for Reader conditions (high traffic, noise, offline, tag capacity) and Zone Manager conditions (offline).  From the Admin Console, you can also set a Global Alert Policy that affects all alerts for all users.
