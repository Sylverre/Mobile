System Attributes

System Attributes are attributes (that is, information about assets and environments) defined by the CenterScape system. System Attributes cannot be edited or deleted as they are integral to CenterScape. The System Administrator can restrict permission to view System Attributes, apply formatting to alter the way each System Attribute is displayed, set whether historical values are recorded, and create additional statistical attributes for select System Attributes.

View System Attributes and Change Selections

  1. Navigate to Admin Console > Data Schema > System Attributes.
  2. Select a System Attribute from the list.
  3. If desired, update selections.
  • Record Value Changes: Check this box if this attribute is required for historical reporting. If unchecked, CenterScape retains only the current value of the attribute; no previous information is available. If checked, all value changes for this attribute for the life of the asset will be recorded for historical reporting and life cycle management.

This box must be checked to enable attribute graphing/reporting directly from the Assets task.

Recording value changes for multiple attributes increases the total size of the database.

  • Restrictable: Check this box if you would like this attribute to be one that can be limited to certain Groups.

Group-based restrictions must be completed in User Access.

  1. If desired, click Add to apply a formatting change to the Attribute that is triggered by a specific value or range.

This is useful for calling attention to values that go out of range, as green for acceptable, yellow to warn, red for alarm.




  1. If adding conditional formatting, in the Add Formatting pop-up window that appears, select foreground color, background color, operator, and value if appropriate. Click OK to return to the Formatting pane. Multiple conditions can be set for a single System Attribute.
  1. Click Save Changes when finished.

See Conditional Formatting for more information.

Statistical Attributes

Statistical Attributes produce derived data in real-time from the raw historical data collected by CenterScape. The statistical computations supported are Maximum, Minimum, Median, Average, and Standard Deviation. Statistical Attributes can be created from any system attributes that report numerical data; the computed values are stored as Asset Attributes and can be used throughout CenterScape in dashboards, graphs, reports, map views, etc.

Statistical Attributes are often used for environmental conditions and power consumption, where trends are of particular importance. You might, for example, create a statistic for the average row intake temperature computed daily using the last thirty days of history. Using the new statistic in Dashboard gauges, show current average temperature vs. the thirty-day average temperature side by side. The dashboard gauges will provide a quick visualization if the current average is outside the norm or trending upwards.

Create a New Statistical Attribute from a System Attribute

  1. Navigate to Admin Console > Data Schema > System Attributes and select an attribute to which you will apply a statistical view.
  2. Click the New Statistic button.

The selected attribute must generate numerical data. The New Statistic button is grayed out if statistical measurements are not relevant for the selected System Attribute.

  1. Complete all required and any desired additional fields for the new Statistical Attribute. Some fields are pre-populated with data inherited from the parent System Attribute.

 

NOTE:  Those fields with asterisks (*) are required fields.

Only fields applicable to the selected System Attribute appear when creating a new Statistical Attribute from a System Attribute.

  • Name: Assign a name to the new Statistical Attribute.
  • Description: If desired, input a detailed comment describing purpose or meaning of the statistical attribute.
  • Record Value Changes: Check this box if this attribute is required for historical reporting. If unchecked, CenterScape retains only the current value of the attribute; no previous information is available. If checked, all value changes for this attribute for the life of the asset will be recorded for historical reporting and life cycle management.
  • Restrictable: Check this box if you would like this attribute to be one that can be limited to certain groups.

Group-based Restriction is completed in User Access.

  • Hide On User Console: Check this box if you would like this attribute to be hidden when viewing assets in the User Console. This provides a way to assign attributes and values to assets as an administrative function.
  • ID: This field is automatically generated by CenterScape when you enter a Name.
  • Type: The data type is inherited from the parent System Attribute and cannot be changed.
  • Asset Reference: Select an asset to link to it.
  • Units: Select the desired unit of measurement.
  • Statistic: Select Minimum, Maximum, Average, Median, or Standard Deviation.
  • Value Constraint: Define the value class allowed. Depending on the system attribute, a statistical attribute can include minimum and maximum value constraints.
  • String Values: Define a list of values for the statistical attribute, and optionally restrict values to that list.
  1. Apply Formatting if desired.
  2. Click Save Changes.

Statistical Attributes can be grouped into Statistical Packs.
