Add Assets to CenterScape

Assets are the physical objects that are managed and/or monitored with CenterScape. You may choose to add assets that will not be tagged as well as tagged assets and assets to which tags have not yet been assigned. Assigning tags to assets can be done as part of, or separately from, the process of adding assets to CenterScape. To audit the location of assets that may be moved, assign a tag to an asset; to collect data from sensors, assign the sensors as assets.

Add a Single Asset

  1. Navigate to User Console > Assets and click New to add an asset manually.
  2. Select the Asset Template or Asset Type from the drop-down menus and complete all required and any desired additional fields in the Create: Asset pop-up window.

 

NOTE:  Those fields with asterisks (*) are required fields.

  1. When completed, click one of the buttons at the bottom of the dialog: Edit Group to restrict access to the asset type, Save & Add Another, or Save. Cancel closes the window without saving.

Assign Tag and Add Single Asset

  1. Navigate to User Console > Tag Management > Manage Tags.
  2. Select the tag(s) under the Import Detected Tags column and click the Add Selected Tag(s) button, or click Add All Tags. This will move the tags to the Unassigned Tags column.
  3. In the Unassigned Tags column, right-click on the tag to be assigned in the far right column and then select New Asset.
  4. Complete the Create: Asset dialog.
  5. When completed, click OK. Cancel closes the window without saving.

Import Assets

RF Code recommends that you create at least one asset using use the appropriate Asset Template or Asset Type, then export these exemplar assets and modify the CSV to import assets.

After you have created one or more exemplar assets:

  1. Navigate to User Console > Assets > Manage Assets.
  2. Click on the new asset to select it. If you have created multiple exemplar assets, ctrl-click to select each of the exemplars.
  3. Click the Export button to export a CSV file from CenterScape to use as a template, or right-click and select Export to export multiple selected exemplars.
  4. Select the Advanced Export option and Export all asset attributes.
  5. Export CSV.
  6. Open the CSV file in a spreadsheet program.
  7. Remove all columns other than:
    8. Class Type guid Retired Deletable $aName $aAsset Tag
    typically Entity. Asset Type leave this field empty. CenterScape will assign a guid on import. For new assets, False. This field describes whether or not a user can delete this asset. Typically True. Assign a unique name for each asset. Enter the unique Tag ID for each tagged asset.
  8. Add information for each new asset in a new row.
  1. Save file as CSV.
  2. Navigate to User Console > Assets > Import Assets.  
  3. Browse to and Upload CSV.

Add Asset Template

Create templates to pre-populate common asset attributes for asset types. Asset Templates minimize the need for data entry, in some cases leaving only Name and Tag to be completed by the user. Asset templates are well-suited to adding assets with use of a barcode scanner.

  1. Navigate to Admin Console > Customization and click Asset Template.
  2. Complete the Template ID and Description fields, and select the desired asset type.
  3. Complete the asset type information fields that appear. Save Changes when finished. The template is now available in the User Console. To view or use, navigate to User Console > Assets > New and click the drop-down arrow for Asset Template.

Edit Asset Template

CenterScape includes Rack and Server asset templates; these assets can be edited, and copied for use as a framework for new templates. Templates you create can also be copied and edited.

  1. Navigate to Admin Console > Customization and click Asset Template.
  2. In the center Hierarchy menu, select the template to edit.
  3. Make the desired changes in the Template Settings pane. Save Changes when finished. The template is now available in the User Console.
