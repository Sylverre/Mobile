Compare Values

Among the functions available for calculated asset attributes are two called in and containsElement that are useful for comparing values.

The in function takes two arguments and evaluates to a boolean:

in(arg1, arg2)

The first argument is a single value to be tested against the second argument.

The second argument can be a single value such as an asset’s name attribute, or a list of values such as a string-list or custom type list.

If we have a location hierarchy of the form:

US

TEXAS

DALLAS

AUSTIN

FLORIDA

MIAMI

ORLANDO

The in function it could be called with the values in(“AUSTIN”, “TEXAS”) and would return true.

In this case, the value of “AUSTIN” is first checked for an exact string match. If that match does not occur, all ancestors of “TEXAS” are checked to see if “AUSTIN” matches any of them.

The containsElement function operates similarly to the in function. However, the ancestors of the value(s) in the second argument are not checked.

The first argument has to have an exact match to one of the values in the second argument.

If we had a schema where we had for example a person and a laptop as two Asset Types:

PERSON

COMPUTER

A boolean calculated asset attribute called PC_IN_EXPECTED_LOCATION could be added to the PERSON type to set a flag to indicate if a PERSON’s LAPTOP is detected to be in the PERSON’s expected location. The expression is:

if ((isNull($aExpectedLocation) || isNull(PC.$aLocation)), null,

in(PC.$aLocation, $aExpectedLocation))

The if condition checks that both the PERSON’s expected location and the PC’s location have values, otherwise the PC_IN_EXPECTED_LOCATION attribute has no value. If both the PERSON’s expected location and the PC’s location have values, the in function is evaluated and set as the value of the calculated asset attribute.
