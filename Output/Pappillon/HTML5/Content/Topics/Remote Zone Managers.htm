Remote Zone Managers Zone Managers

A local Zone Manager is installed as part of CenterScape᾽s initial installation. This Zone Manager can handle 1,000 readers and 100,000s of tags if installed on a powerful server system. The number of tag events has a far larger impact on the resource requirements of the Zone Manager system(s) than the number of readers. The need for additional remote Zone Managers can be driven by two factors:

  • Scalability: The need to handle more tags and readers and spread the load over additional systems.
  • Network Architecture and Connectivity: The need to distribute Zone Managers to remote locations in a distributed environment.

To set up CenterScape with additional Zone Managers, first install those Zone Managers on the desired servers by running the CenterScape installer EXE, selecting the option to install only the Zone Manager component. Then log in to CenterScape.

  1. Navigate to Configuration > Zone Managers and click New.

 

  1. Complete all required and any desired additional fields.

NOTE:  Those fields with asterisks (*) are required fields.

Basic Information

  • Name: Input a unique ID for this new Zone Manager.
  • Description: If desired, add a detailed comment describing the new Zone Manager.
  • Enabled: Check this box to allow the new Zone Manager to go online once saved.

Direct Network Settings

  • Hostname: Enter the IP address of your new Zone Manager in this field.
  • Port: Enter the port number over which to communicate with your new Zone Manager.
  • SSL: Select the desired SSL mode:
    • OFF: turns SSL mode off on the Zone Manager
    • IFAVAIL: uses SSL on the Zone Manager if available
    • REQUIRED: requires use of SSL
    • STRICT: will authenticate the matching hostname and only connect if the hostname matches the Zone Manager

Up Connect Settings

When this option is configured, the Zone Manager initiates the contact with the CenterScape rather than requiring that access be granted to allow CenterScape to connect to Zone Managers behind a firewall. Up Connection settings must be specified in the configuration of your remote Zone Manager prior to configuring this Zone Manager for use with CenterScape. The Up Connection Zone Manager ID and password are obtained in the configuration of the remote Zone Manager.

  • Up Connection Zone Manager ID: Enter the ID that was assigned when you configured your Zone Manager for Up Connection.
  • Up Connection User ID: Enter the User ID that was assigned when you configured your Zone Manager for Up Connection.
  • Up Connection Password: If you assigned a password to your Zone Manager when you configured it for Up Connection, enter the same password here.
  • Confirm Password: Enter the password again to confirm it.

User Account

Set up a user account if desired by entering a username and password to allow for administrator or view-only access of the Zone Manager.

  • Administrator Username: Create an administrator ID.
  • Administrator Password: Enter a password for the administrator ID.
  • Confirm Password: Enter the password again to confirm it.
  • Read-only Username: Create a user ID with read-only access to the Zone Manager.

Advanced

  • Offline Timeout: The number of seconds since the last successful message before readers are reported as offline.
  • Expected Location Bias: Increases the confidence level of a location assertion; required in order to tune reader zones but not otherwise used.
  • Initialization Delay: Set a period to delay initialization of the new Zone Manager to prevent conflicts with other items competing for the same resources.

IBM® WebSphere® Sensor Events

These fields are used specifically for configuring Zone Manager for use with IBM WebSphere. This configuration section is optional and not required for basic configuration of a Zone Manager for use with CenterScape.

  1. Click Save Changes. The newly configured Zone Manager appears in the Zone Manager list.

To configure CenterScape to interact with more than one Zone Manager, repeat these steps for each Zone Manager.
