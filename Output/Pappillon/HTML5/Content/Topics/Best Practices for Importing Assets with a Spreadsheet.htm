Best Practices for Importing Assets with a Spreadsheet

System administrators may choose to use a spreadsheet to import and configure certain settings for a particular group of assets that will be used in the CenterScape system. This method is particularly useful if you need to import several assets of the same asset type. The process involves customization of a CSV file where tag IDs and asset attributes are configured to an administrator's particular requirements.

The following is an example of best practices for importing assets with the use of a CSV spreadsheet:

Ensure that there is at least one asset configured in your system that is the same asset type that you will be configuring using the spreadsheet importing method.

Create a custom view for the asset types that you will be importing with the spreadsheet by navigating to the User Console > Customization > Views and click on the New button. Enter a Name and Description (optional) for the View. Select the asset type(s) that you will be importing into your system. Select the required attributes for the Import CSV file creation for this example [Name, Asset Tag, Asset Type, Lock Location, and any additional asset attributes you would like to configure with the CSV spreadsheet for the set of assets you will be importing (optional)]. Click the Save Changes button to save the asset view.

Figure 209: Asset Manager Custom View Creation

2. Navigate to Assets > Manage Assets By Type and select the custom view that you just created from the views drop-down list in the top, right-hand corner of the screen. The view will change to your custom view and you will see the asset(s) that you configured in the view. Click the Export button and select "Export asset attributes in view". Click the Export CSV button.

Figure 210: Export CSV File of Custom View

Open the CSV file in Microsoft Excel or other software to edit the CSV file. Delete the columns titled "retired" and "deletable". Enter the information for the assets for each of the required fields (class, type, guid, $aLocation, $aName, $aAssetTag, $aLockLocation for this example) that you are going to add to the CenterScape system. Also input a value for your specified custom attributes (optional). Please note the following tips for the populating of attribute columns:

TIP: The fields that are required to have a value specified in the import CSV file will depend upon whether that field is required for the asset types of the assets you will be configuring using the CSV file. You can determine which fields are required for a particular Asset Type by accessing the CenterScape Administrator Console > Data Schema > Asset Types, selecting the desired Asset Type from the asset tree and clicking the View Sample Input Form button. The sample input form will appear for that particular Asset Type. Within that sample input form there should be one or more fields with a red asterisk (*) in front of it. This indicates that it is a required field for the Asset Type and therefore, that field must be included in the CSV file and must have a value for the import of the CSV file to be successful in creating the assets in the CenterScape system.

All entries for the "class" column will be "entity" for every asset that will be configured in the CSV file.

The entry values for the "guid" column can be found by locating the Asset Type or Attribute (title of the column) in the Administrator Console > Data Schema > Asset Attributes or Administrator Console > Data Schema > Custom Attribute Types sub-tasks in the ID field. Type the guid value from the ID field into the required field in the CSV file for the Asset Type.

Each entry for $aName field must be a completely unique value. You cannot import two assets with the same Name.

Figure 211: Example CSV Import File Configured for Multiple Asset Creation

In this example we will be adding a mixture of Inventory Assets and Sensor Assets. These Asset Types are derived from the data schema. You will notice in the figure above that the type column has both SERVER and HUMMIDITY_TEMPERATURE Asset Types.

The guid field has a value for the asset that was exported from CenterScape. This was be automatically generated by the CenterScape. If the cell values for this column are left blank, the CenterScape will generate a unique guid for the asset upon import.

The $aLocations field value has been automatically generated for the exported asset by CenterScape. If the Locations field values are left blank in the file, CenterScape will automatically assign a Location to the asset based on the Locations/Rules that have been configured in CenterScape. If no rules apply to the asset, then the asset will be assigned to "Unknown Location". This works well for Inventory Asset Types because these assets will likely be mobile assets in an CenterScape system that dynamic location data will be desired for tracking location. However, for Sensor Asset Types, these are generally stationary locations because they have been installed in a specific place (such as a room, rack or closet) to monitor the environmental conditions of that area. So a static location will need to be configured. In CenterScape this is done through a specification of the sensor's static location and by setting the $aLockLocation field value to true. Meaning that the sensor is in a set stationary location and will not move. So for this example, you will see in the above figure that a value has been assigned for the Location fields for the two sensor assets and that the Lock Location fields have been set to the value of TRUE.

Each entry for $aName field must be a completely unique value. Notice in the figure above that for each asset the Name field value is unique. The import to the CenterScape system will fail and prompt an error message if two or more assets are present in the CSV file with the same Name.

For the $aAssetTag field values, you need to assign an Asset Tag ID to each of the assets that you will be creating. The Asset Tag ID consists of the Group Code and Tag ID that are present on the label of each tag that you will be associating to an asset.

Figure 212: Group ID, Tag ID, and Treatment Code Location

Once you have completed the Import CSV file configuration, save the file. Navigate to User Console > Assets > Import Assets and browse and select the file for import. Click the Upload button.

Figure 213: Import CSV File to Create Assets

The configured assets will now appear in your CenterScape system.
