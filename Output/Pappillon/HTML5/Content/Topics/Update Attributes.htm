Update Calculated Attributes

Among the calculations that can be performed is “update,” used to set the value of a calculated attribute whenever certain user-specified criteria are met. The update () function can take either two or three arguments.

When calling an update with two arguments, the form is:

update(“target attribute”, value)

The first argument is the name of the attribute whose value is to be updated, enclosed in quotation marks. Note that, unlike other functions where an attribute is referenced and the value of that attribute is inserted into the expression, the value of the attribute is the actual name of the attribute itself.

For example, say we have an String attribute called OWNER_NAME. If it is desired to use the update function to change the value of OWNER_NAME, the syntax would be:

update(“OWNER_NAME”, “BOB”)

This would update the NAME attribute on the asset which contains the calculated attribute.

If calling with three arguments, the first argument is expected to be an asset reference attribute. The form would be:

update(asset reference attribute, target attribute, value)

In an example where there is a schema that has a person and a laptop which is assigned to the person, the schema would have the following asset types:

PERSON

COMPUTER

An asset reference attribute called LAPTOP whose asset reference type is COMPUTER would then be created and assigned to the PERSON asset type.

Next, an asset attribute would be created that would update the COMPUTER asset’s OWNER_NAME attribute whenever a COMPUTER asset is chosen as the value of a PERSON asset's LAPTOP attribute.

update(PC, “OWNER_NAME”, $aName)

This example can then be extended to assert that the LAPTOP assigned to a PERSON is expected to be in the same location as the PERSON. The following calculated attribute uses the update() function to set the COMPUTER asset's expected location to the current location of the PERSON:

update(PC, “$aExpectedLocation”, $aLocation); true

Please note that all attributes are required to have a “Type.” Since this calculated attribute’s type is “Boolean” we use the expression “true” after updating the attribute because the value of a calculated attribute is the last value found when evaluating the attribute's expression. If this calculated attribute were viewed in the user interface or in a report, the value “true” would be displayed as the attribute’s value. For this reason it is often wise to hide the attribute by selecting “Hide On User Console” in the calculated attribute’s configuration.
