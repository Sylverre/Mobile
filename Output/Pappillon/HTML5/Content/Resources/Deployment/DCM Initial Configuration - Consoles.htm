DCM: Initial Configuration

Added text from the AM User Manual to add to the original Deployment Guide (AM section Configuring and Using Asset Manager)--because my first impression on reading Deployment was that I'd missed something, though it did become clear eventually that this was where I-the-user would first land after installation.

After installing CenterScape, the first things you need to do are basic configurations. This task is accomplished through the web interface. The five (5) preliminary components that you need to configure are: a Database, a Zone Manager, a Reader, Tag Groups, and a basic License Key.

The CenterScape Database and a local Zone Manager will already be configured if you chose the local Zone Manager option and the SQL Server Express options during the installation process. These components will display as configured, and no further action need be taken.

When you first log in to CenterScape, the interface opens to the Admin Console > Dashboard > Configuration.

Consoles

There are two primary views (consoles) in CenterScape, accessible at any time:

  • Admin Console: Access this console by clicking the Admin Console link in the lower right corner of the web interface of CenterScape. This console is typically used for administrative functions of the server and where a large portion of this document focuses. It could be considered the "Control Panel" of your CenterScape. This is where most of the initial configuration is done.
  • User Console: Access this console by clicking the User Console link in the lower right corner of the web interface of CenterScape. This console is for managing and viewing sensors, alerts, dashboards, reports, graphs, etc. This is where the final configuration and daily use of CenterScape is done.

NOTE: Switching between the two consoles is done by clicking either the Admin Console or the User Console link at the bottom right of the CenterScape user interface. The two links toggle depending on which console you are using.
