Thresholds and Alerts

More properly SysAdmin than Deployment, but could be argued either way. Not yet removed from TOC, since personnel completing physical deployment may be more aware of thresholds, but I tend to feel that anything involving the User Console is beyond the scope of the Deployment Guide.

Thresholds are limits for specific attribute values. Alerts are notifications you will receive if the value of a particular attribute goes beyond the threshold set for it. For example, you can set a threshold for a sensor tag’s temperature reading to be 77°F and have an alert sent when the temperature exceeds this threshold.

To configure a basic threshold and alert, perform the following steps:

1. In the User Console, go to Alert Management.

2. Click Actions, click the New button, and choose Email Alert Action from the drop-down menu.

3. Give the alert a name, enter the email address of the alert recipient, and then click Save Changes.

4. Select Temperature Alert Threshold from the dropdown list and fill in a Name for the alert, e.g. “Rack too hot” (see screenshot below).

5. In the Alert Condition section, set the Threshold Attribute Value Operator to be Greater Than (>) and the Threshold Attribute Value to 80.

6. Set the Alert Filter so that all temperature sensors in the RF Code Data Center are monitored. This is a convenient way of setting a threshold on multiple sensors at one time for a given location, which will result in you receiving an alert anytime a sensor reads a temperature that is greater than the threshold you set.

7. If you choose the Email Alert Action, then an email like the one below will be sent when any temperature sensor in a particular data center location (e.g., RF Code Data Center) reads greater than 80 degrees.

Alert thresholds and email notifications can be set up for other sensors such as humidity, fluid detection, and/or open door states.
